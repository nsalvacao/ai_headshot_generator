Adoption scenarios

Personal and professional contexts addressed

Each use case maps to explicit prompt templates and wardrobe presets so that creative direction remains consistent across teams.

Individual identity refresh

Generate LinkedIn or conference portraits that respect the subject’s identity while updating wardrobe, lighting and framing. The client enforces base64 validation and MIME gating before any pixel leaves the browser.

  • Preset: Professional Headshot / Classic B&W
  • Fine-tune sliders for subtle smile & gaze alignment
  • Local preview + deterministic filenames for exports

Talent & HR enablement

Teams can request consistent avatars for onboarding portals or campaign assets. Batch prompts are composed via the Wardrobe modal and submitted sequentially to respect Gemini rate limits.

  • Shared presets via wardrobe.ts metadata
  • Model configuration stored in localStorage with explicit consent
  • Status overlays distinguish loading, success and retries

Creative studios & agencies

Studios can point the UI at internal inference clusters (Ollama or proprietary services) using the Custom provider. HTTP timeouts, schema validation and friendly error mapping safeguard operators.

  • Custom API URL + model name per environment
  • Composable prompts with aspect-ratio guardrails
  • Download-ready PNG output for downstream editors
Architecture

Composable client-side orchestration with explicit gateways

The solution optimises for transparency: each module has a single responsibility and can be unit tested without invoking external inference.

Capture & preparation

StartScreen validates image types, converts to Data URLs and stores them in React state only. No persistence or third-party uploads happen until the user explicitly triggers generation.

Prompt synthesis

WardrobeModal assembles structured prompts by combining wardrobe metadata, aspect ratio tokens and fine-tune adjustments. The resulting string is versioned for reproducibility.

Inference gateway

geminiService and customModelService translate inputs into the expected provider schemas, enforce MIME parsing and propagate descriptive errors back to the UI layer.

Post-processing & delivery

Responses are normalised into GeneratedImage contracts, surfaced in the gallery grid, and made downloadable with deterministic filenames to support asset catalogues.

Execution pipeline

From user intent to rendered portrait

The flow below highlights the control points where validation, logging and governance decisions occur.

1 · Input validation

Client enforces MIME presence, base64 payloads and image type restrictions before persisting anything in memory.

Security control: rejects malformed Data URLs immediately.

2 · Prompt curation

Wardrobe metadata + fine-tune deltas produce a deterministic prompt, ensuring identical requests can be reproduced for audits.

Governance: style catalog maintained in wardrobe.ts.

3 · Model invocation

Gemini gateway signs each request with the user-provided API key; custom mode wraps HTTP POST to arbitrary inference servers.

Observability: descriptive errors surface back to users.

4 · Delivery & export

Base64 image data is rendered, optionally downloaded, and cleared when the user resets the session.

Data retention: nothing stored server-side by default.

Runtime configuration

Users bring their own API keys. Configuration is client-side: localStorage stores provider choice, API key, custom URLs and model identifiers.

  • Gemini key stored only in browser storage
  • Custom backends use HTTPS POST with JSON payload
  • Error helper (lib/utils.ts) standardises user messaging
